## Test Driven Development (TDD) Project
This projects follows the TDD methodology, which means to create first the tests, thinking in the possible features of the function you want to create. Furthermore, it is important to create the "Edge Cases of your function" during the creation of your tests.<br/><br/>
TDD methodology will help you to implement a robust function. Once you have already implemented your tests, then you have to create your functions.<br/><br/>
Also, please not forget to export your functions in the file where you are implementing the functions. Moreover, in the test file please link the file of your functions. <br/><br/>
<strong>Classical Cycle:</strong> <br/>
Red : When you are writing the test.<br/>
Green : When you implement the function.<br/>
Blue: This is the refactor, the moment when we clean the code, without breaking the tests.<br/>
